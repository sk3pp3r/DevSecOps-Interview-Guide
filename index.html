מה זה DevSecOps?

DevSecOps היא גישה לשילוב שיטות אבטחה בכל שלבי מחזור חיי פיתוח התוכנה (SDLC), החל מתכנון, קידוד, בדיקה ושחרור, ועד לפריסה ותפעול. המטרה היא להטמיע אבטחה כחלק אינטגרלי מהתרבות והתהליכים של DevOps, במקום להוסיף אותה כשלב מאוחר (Shift Left Security).

מה כולל DevSecOps?
  • אוטומציה של אבטחה: שילוב בדיקות וכלי אבטחה אוטומטיים ב-CI/CD pipeline.
  • אבטחה כקוד (Security as Code): הגדרת מדיניות ובקרות אבטחה בקוד.
  • ניטור רציף: מעקב אחר אירועי אבטחה, פגיעויות ושינויי תצורה בזמן אמת.
  • ניהול פגיעויות: זיהוי, תיעדוף ותיקון חולשות באפליקציות ובתשתית.
  • ניהול סודות (Secrets Management): אבטחה ואוטומציה של גישה למפתחות, סיסמאות ותעודות.
  • הגנה על תשתית ענן (Cloud Security): אבטחת סביבות ענן, כולל תצורה, זהויות ורשת.
  • תגובה לאירועים: תהליכים מוגדרים לטיפול מהיר ויעיל באירועי אבטחה.
  • תרבות שיתוף פעולה: עידוד תקשורת ושיתוף ידע בין צוותי פיתוח, אבטחה ותפעול.
תחומי אחריות עיקריים של DevSecOps Engineer
  • הטמעת כלי סריקת קוד (SAST, DAST, SCA) ב-pipeline.
  • אבטחת קונטיינרים וסביבות Kubernetes.
  • עבודה עם כלים לניהול תצורת אבטחה בענן (CSPM).
  • הגדרת ומעקב אחר מדיניות אבטחה אוטומטית (Policy as Code).
  • ביצוע בדיקות חדירות וסקרי סיכונים.
  • הדרכה והעלאת מודעות אבטחה בקרב צוותי פיתוח.
  • השתתפות בתכנון ארכיטקטורות מאובטחות.
  • פיתוח סקריפטים וכלי אוטומציה לאבטחה.

📚 מושגי יסוד

SAST

Static Application Security Testing - סריקת קוד סטטי לזיהוי פרצות אבטחה לפני הרצת הקוד

DAST

Dynamic Application Security Testing - בדיקות אבטחה על אפליקציה רצה בזמן אמת

SCA

Software Composition Analysis - סריקת dependencies וספריות חיצוניות לפרצות אבטחה

CSPM

Cloud Security Posture Management - ניהול נוף האבטחה בענן

DSPM

Data Security Posture Management - ניהול ואבטחת נתונים בענן

Zero Trust

מודל אבטחה שמניח שכל גישה היא חשודה עד הוכחת היפך

Shift-Left Security

גישה המשלבת אבטחה בשלבים המוקדמים של פיתוח התוכנה, במקום לטפל בבעיות בסוף התהליך.

Policy as Code

הגדרת מדיניות אבטחה בקוד, מה שמאפשר versioning, testing ואוטומציה של כללי אבטחה.

🔧 שאלות טכניות

איך היית מגדיר CI/CD pipeline מאובטח?

תשובה מומלצת:

שלב התכנון: משלב security checks בכל שלב של הpipeline

  • Source Code Analysis: SAST tools לסריקת קוד
  • Dependency Scanning: SCA tools לבדיקת ספריות
  • Container Scanning: סריקת Docker images
  • Infrastructure Scanning: בדיקת Terraform/CloudFormation
  • Dynamic Testing: DAST על staging environment
  • Secrets Detection: מניעת חשיפת credentials
  • Compliance Checks: וידוא עמידה בתקנים
איך מגדירים monitoring מתקדם ב-AWS?

כלים עיקריים:

  • CloudTrail: מעקב אחר כל API calls
  • GuardDuty: זיהוי איומים באמצעות ML
  • Security Hub: מרכז ניהול ממצאי אבטחה
  • Config: מעקב אחר שינויי תצורה
  • VPC Flow Logs: ניתוח תעבורת רשת
  • CloudWatch: metrics ו-alerting
מה האתגרים באבטחת Kubernetes?

אתגרים עיקריים:

  • Pod Security: הגדרת Pod Security Standards
  • Network Policies: הגבלת תקשורת בין pods
  • RBAC: ניהול הרשאות גרנולרי
  • Image Security: סריקה וחתימה של container images
  • Runtime Security: מעקב אחר התנהגות בזמן ריצה
  • Secrets Management: אבטחת credentials ומפתחות
איך מבטיחים אבטחה ב-Infrastructure as Code?

שכבות אבטחה:

  • Static Analysis: Checkov, Terrascan, tfsec
  • Policy Enforcement: Open Policy Agent (OPA)
  • Code Review: בדיקה ידנית של שינויים
  • State Security: הצפנת Terraform state
  • Version Control: Git flow עם branch protection
  • Automated Testing: unit tests לתשתית

🤝 שאלות התנהגותיות

תאר סיטואציה שבה זיהית סיכון אבטחה משמעותי

מבנה STAR:

Situation: תאר את ההקשר והסיטואציה

Task: מה היה התפקיד או המטלה שלך

Action: איזה פעולות נקטת

Result: מה התוצאות והלקחים

דוגמה לתשובה טובה

Situation: "זיהיתי שצוות הפיתוח משתמש בהרשאות Admin בסביבת הייצור..."

Task: "הייתי צריך להפחית את הסיכון מבלי לפגוע בפרודקטיביות..."

Action: "יצרתי IAM roles ספציפיים, הדרכתי את הצוות, יישמתי MFA..."

Result: "הפחתנו את המשטח התקיפה ב-80% תוך שמירה על זמני deployment..."

איך אתה משכנע developers לאמץ security practices?

אסטרטגיות:

  • הסבר החשיבות העסקית: קשר לתוצאות עסקיות
  • הקל על התהליך: ספק כלים ואוטומציה
  • הדגם בדוגמאות: הראה מקרי test מהעבר
  • תן הכשרה: השקע בלימוד הצוות
  • צור תרבות: עודד sharing וטיפים

🎨 ספציפי

איך תתמודד עם אבטחת פלטפורמה של מיליארדי אירועים יומיים?

אסטרטגיית סקלה:

  • Automation First: אוטומציה של כל תהליכי האבטחה
  • Real-time Monitoring: זיהוי אנומליות בזמן אמת
  • Microservices Security: אבטחה ברמת כל שירות
  • API Security: Rate limiting, OAuth, WAF
  • Data Classification: הגנה מותאמת לרגישות המידע
  • Incident Response: תגובה מהירה לאירועים
איך תבטיח Compliance בסביבה גלובלית?

אתגרי Compliance:

  • GDPR: הגנה על נתונים של משתמשים אירופאים
  • CCPA: תקנות קליפורניה לפרטיות
  • SOC 2: ביקורת אבטחה ופרטיות
  • ISO 27001: תקן ניהול אבטחת מידע
  • PCI DSS: אבטחת עסקאות כרטיסי אשראי
אתגרים ייחודיים
  • Marketplace בינלאומי עם חוקים שונים
  • עסקאות כספיות הדורשות אבטחה גבוהה
  • מידע רגיש של freelancers ולקוחות
  • צורך באמינות גבוהה (99.9% uptime)

  • עדכן את עצמך בטכנולוגיות העדכניות

במהלך הראיון
  • הראה חשיבה מערכתית ופתרון בעיות
  • דגיש ניסיון עם automation וscaling
  • תן דוגמאות של שיתוף פעולה עם developers
  • הראה למידה מתמשכת ועניין בטכנולוגיה
  • שאל שאלות על התרבות והאתגרים
מה לא לעשות

  • לא להיות טכני מדי עם אנשים לא טכניים
  • לא לבקר את המקום הנוכחי שלך
